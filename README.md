# DonJuan VPN

A lightweight, self-contained web UI manager for [sing-box](https://sing-box.sagernet.org/). Dynamically generates sing-box configurations, manages proxy lifecycle, handles TUN routing, and provides rule management from a single binary.

## Supported Protocols

- **vless** (transports: none, tcp, grpc, httpupgrade)
- **hysteria2 (hy2)**
- **trojan** (transports: none, tcp, grpc, httpupgrade)

## Build

### Prerequisites

- Go 1.21+

### Required Source Files

```
singbox_ui/
├── main.go        # HTTP server, handlers, startup
├── parser.go      # Config generation, protocol parsers
├── process.go     # sing-box process management
├── models.go      # Data structures
├── index.html     # Frontend (embedded in binary)
├── go.mod         # Go module definition
└── go.sum         # Dependency checksums
```

All 7 files are required to build. `go.sum` is generated by `go mod tidy` but must be present.

### Build for your x86 laptop

```bash
cd singbox_ui
go build -ldflags="-s -w" -o donjuan
```

### Cross-compile for OpenWrt (arm64 cortex-a53)

From your x86 laptop:

```bash
cd singbox_ui
CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build -ldflags="-s -w" -o donjuan
```

Then copy the binary to your router:

```bash
scp -O donjuan root@<ROUTER_IP>:/usr/bin/donjuan
```

> `CGO_ENABLED=0` is essential — it produces a fully static binary with no glibc dependency. OpenWrt uses musl libc, and a dynamically-linked binary will not work. Use `scp -O` because OpenWrt lacks the sftp subsystem.

This produces a single static binary `donjuan` with the frontend embedded. **Go is only needed at build time.** The compiled binary has no runtime dependencies.

## OpenWrt Setup

### 1. Install kernel modules

```bash
opkg update
opkg install kmod-tun kmod-nft-core kmod-nft-nat kmod-nft-bridge nftables
```

### 2. Install sing-box

Download the arm64 binary from [sing-box releases](https://github.com/SagerNet/sing-box/releases):

```bash
# Find the latest version at https://github.com/SagerNet/sing-box/releases
# Example for v1.13.11:
wget https://github.com/SagerNet/sing-box/releases/download/v1.13.11/sing-box-1.13.11-linux-arm64.tar.gz
tar xzf sing-box-1.13.11-linux-arm64.tar.gz
cp sing-box-1.13.11-linux-arm64/sing-box /usr/bin/sing-box
chmod +x /usr/bin/sing-box
```

### 3. Install DonJuan binary

```bash
scp -O donjuan root@<ROUTER_IP>:/usr/bin/donjuan
chmod +x /usr/bin/donjuan
```

### 4. Create data directory and run

```bash
mkdir -p /etc/donjuan/data
cd /etc/donjuan
donjuan
```

Or create a procd init script at `/etc/init.d/donjuan`:

```bash
#!/bin/sh /etc/rc.common
START=99
STOP=10
USE_PROCD=1

start_service() {
    procd_open_instance
    procd_set_param command /usr/bin/donjuan
    procd_set_param respawn
    procd_set_param stdout 1
    procd_set_param stderr 1
    procd_set_param env HOME=/etc/donjuan
    procd_set_param limits nofile=65535 65535
    procd_close_instance
}
```

```bash
chmod +x /etc/init.d/donjuan
/etc/init.d/donjuan enable
/etc/init.d/donjuan start
```

### 5. Configure DonJuan

1. Open **http://<ROUTER_IP>:8888** in your browser
2. Enable **TUN** and **Auto Redirect** in Core Settings
3. **Auto Redirect** uses nftables to redirect LAN traffic through the TUN interface — this is required on OpenWrt and works automatically with fw4

### 6. Split routing (optional)

To route traffic from different LAN interfaces differently, set up separate networks in OpenWrt:

- `br-lan` (e.g. 10.1.1.0/24) — "vpn" WiFi, traffic goes through proxy
- `br-novpn` (e.g. 10.1.2.0/24) — "novpn" WiFi, traffic bypasses proxy

Then add a custom rule in DonJuan:

| Type | Value | Action | Source |
|---|---|---|---|
| IP | 0.0.0.0/0 | Direct | 10.1.2.0/24 |

This generates `source_ip_cidr: ["10.1.2.0/24"]` in the sing-box config — all traffic from the "novpn" subnet bypasses the proxy.

## Run (non-OpenWrt)

### Prerequisites

- Linux (Debian, Ubuntu, RedHat)
- [sing-box](https://sing-box.sagernet.org/) installed on the system
- Root privileges (required for TUN interface and process management)

### Required Runtime Files

```
./donjuan          # The compiled binary (only file needed)
```

Everything else is created at runtime:

| File | Created When | Purpose |
|---|---|---|
| `data.json` | First run | UI state, server configs, routing rules, language preference |
| `router.json` | First OpenWrt run | OpenWrt specific state (WiFi networks, passwords) |
| `config.json` | On start proxy | Generated sing-box configuration |
| `data/` | On geodata download | Cached `.srs` rule sets |
| `/tmp/*.log` | When save logs is ON | Temporal log files to avoid flash wear |

### Start

```bash
sudo ./donjuan
```

Open **http://127.0.0.1:8888** in your browser.

### Port

Default port is `8888`. Change it in **Settings > Advanced** then restart.

## DNS Architecture

DNS resolution uses two servers:

- **remote-dns** (`8.8.8.8`, UDP) — resolves via the proxy tunnel by default
- **direct-dns** (`223.5.5.5`, UDP) — resolves proxy server hostnames and direct-bound domains directly

When TUN is active, all DNS traffic is hijacked by sing-box and routed through the proxy, preventing ISP DNS interference. Proxy server hostnames resolve via direct-dns to avoid circular dependencies.

## Multi-Language

Supported languages: English, Español, 中文, Русский.
Select in **Settings > Advanced**. The choice is saved in `data.json` and persists across restarts.

## FAQ

**Do I need Go installed to run DonJuan VPN?**
No. The `donjuan` binary is statically compiled and self-contained. Go is only needed if you want to build from source.

**How do I update sing-box?**
Go to footer, if an update is available click the version link.

**Why doesn't TOR work on a fresh install?**
Make sure these defaults are ON in Core Settings: **TUN**, **Fake IP**, **Sniffing**, **Local Network**. These are enabled by default. If they were turned off at any point (saved in `data.json`), delete `data.json` and restart to restore defaults.

**Do I need `kmod-nft-tproxy` on OpenWrt?**
No. DonJuan uses sing-box's `auto_redirect` feature with TUN mode, which is the recommended approach since sing-box 1.10+. It provides better routing, higher performance than tproxy, and automatically inserts compatibility rules into OpenWrt's fw4 table.

**`scp` gives `ash: /usr/libexec/sftp-server: not found`?**
Use `scp -O` to use the legacy SCP protocol, which OpenWrt's busybox supports.
